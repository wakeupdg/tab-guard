Tab Guard Adversarial Test

Setup: Arm this tab in Tab Guard before running these tests. These scenarios test indirect/adversarial methods to bypass the JS-layer enforcement, plus special cases for data: and blob: URL handling.

Scenario 1: Clickjacking Overlays

Full-screen transparent/near-transparent fixed-position overlay that attempts window.open() on user interaction.

Scenario 2: Timer-based Popups

window.open() triggered via setTimeout, not tied to direct user interaction.

Scenario 3: mouseleave-triggered Popups

window.open() triggered when mouse leaves page boundary.

Move your mouse out of this red box to trigger window.open() to mismatched URL

Scenario 4: Tab-under Hijack

Opens a decoy tab (same URL as current page) then redirects the original/armed tab to an evil URL via window.location.replace().

Implementation note: This is caught by Location.prototype.replace hook in main-world-blocker.js, gated on same-ORIGIN check (blocks cross-origin page-script navigation). The hook catches origin mismatches without restricting legitimate user navigation (address bar, real link clicks).

Also defended against: meta[http-equiv=refresh] tags are stripped while armed by content-bridge.js (declarative navigation mechanism).

Scenario 5: about:blank + document.write() Injection

Opens a blank window and injects attacker-controlled HTML via document.write().

Expected behavior: Window should remain OPEN (not closed), per the "Null Origin (Allowed)" exemption for benign document.write patterns like Print Invoice.

Scenario 6: Full-screen iframe Overlays

Creates a full-screen iframe overlay pointing to a mismatched URL. Note: iframe (sub_frame) blocking is NOT covered by DNR per the plan (to avoid breaking Stripe/YouTube/reCAPTCHA).

Scenario 7: Tracked Child navigating to data: URL

Opens a child window, then navigates it to a data:text/html,... URL. Should be CLOSED (treated as mismatched origin per existing mismatch logic).

Scenario 8: Tracked Child navigating to blob: URL

Opens a child window, then navigates it to a blob: URL generated from a Blob object. Should NOT be CLOSED (blob: URLs are exempt per OI-003 exemption).
